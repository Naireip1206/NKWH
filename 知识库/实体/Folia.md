# Folia

> 由 Paper 团队开发的 Region-Based 多线程 Minecraft 服务端核心，首次让 MC 服务端真正利用多核 CPU。

## 概述

Folia 是 Minecraft Java Edition 的一个服务端核心，由 Paper 团队（Pl3xMC）开发。它与所有传统服务端核心最大的不同在于**架构层面的突破**：不再使用单线程主循环，而是将世界划分为多个 Region（区域），每个 Region 可以独立地在不同线程上运行 Tick 循环。

## 关键信息

| 属性 | 内容 |
|------|------|
| **类型** | Minecraft 服务端核心（基于 Paper） |
| **API 生态** | Bukkit / Spigot / Paper 插件（**部分不兼容**） |
| **架构** | Region-Based Multi-Threading（基于区域的多线程） |

| **源码仓库** | [GitHub - PaperMC/Folia](https://github.com/PaperMC/Folia) |

## 架构创新

### 传统单线程架构（Paper / Purpur）

```
单线程： [世界1 → 世界2 → 世界3 ...]
所有世界的 Tick 在同一个线程上串行执行。
多核 CPU 大部分时候处于空闲状态。
```

### Folia 多线程架构

```
线程1： Region A（世界1 的区块 [0,0] ~ [31,31]）
线程2： Region B（世界1 的区块 [32,0] ~ [63,31]）
线程3： Region C（世界1 的区块 [0,32] ~ [31,63]）
线程4： 世界2（整个世界的实体和红石）
线程5： 世界3
...并行执行...
```

### 核心概念：Region

- 世界被划分为多个 **Region**（区域）
- **玩家聚集区**：每个玩家及其周围的区块组成一个独立 Region
- **无人区**：没有玩家的区域共享更少的线程资源
- Region 之间通过**消息队列**进行跨区域通信

## 优势

- 真正利用多核 CPU，多世界和大世界的性能显著提升
- 高密度实体场景不会阻塞整个服务器的 Tick
- 大型服务器（数十到上百玩家）的承载能力远超传统核心

## 局限性

- **插件兼容性问题** — 很多插件假设线程模型是单线程的，在 Folia 上无法正常运行
- **小服无优势** — 个位数玩家的服务器，单线程已经足够，Folia 只增加复杂度
- **Region 边界问题** — 跨 Region 的实体交互和红石机械需要特殊处理
- **文档和教程相对较少** — 社区积累不如 Paper 丰富

## 适用场景

- **大型生存服**（50+ 玩家同时在线）
- **多世界架构的服务器网络**
- **高密度实体或大规模红石机械场景**
- **对多核利用率有极致需求的场景**
