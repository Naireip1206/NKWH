# Fabric

> 轻量级 Minecraft Mod 加载器，以启动速度快、API 简洁、更新迅速著称。

## 概述

Fabric 是 Minecraft Java Edition 的一个 Mod 加载器（Mod Loader），与传统的 Forge 相比，它采用更轻量、更模块化的设计哲学。Fabric 本身只提供最基础的 Mod 加载能力，配合 **Fabric API** 提供可选的事件系统和接口。这种设计使得 Fabric：
- 启动速度远快于 Forge
- 更容易跟上 Minecraft 的版本更新
- 安装和配置更简洁
- 按需加载，不需要的接口可以不引入

## 关键信息

| 属性 | 内容 |
|------|------|
| **类型** | Mod 加载器 |
| **首次发布** | 2018 年（1.14 时期） |
| **主要对手** | Forge / NeoForge |
| **API** | Fabric API（可选，轻量钩子） |
| **维护状态** | ✅ 活跃开发中 |
| **官方网站** | [fabricmc.net](https://fabricmc.net) |
| **源码仓库** | [GitHub - FabricMC](https://github.com/FabricMC) |

## Fabric vs Forge

| 维度 | Fabric | Forge / NeoForge |
|------|--------|------------------|
| 架构 | 轻量钩子注入 | 大规模事件系统 |
| 启动速度 | 快 | 慢（大量 Mixin 处理） |
| 版本跟进 | 极快（数天内适配新版本） | 较慢 |
| Mod 规模 | 中小型 Mod 为主 | 大型整合包为主 |
| 学习曲线 | 低 | 较高 |
| 优化模组生态 | 丰富（Lithium/Starlight 等） | 较少 |

## Fabric + 优化模组方案

Fabric 搭配专门的优化模组是一种被广泛推荐的服务端搭建方案。核心优化模组包括：

| 模组 | 作用 |
|------|------|
| **Lithium** | 通用计算逻辑优化，减少不必要的计算开销 |
| **Phosphorus / Starlight** | 光照引擎优化，大幅降低光照卡顿 |
| **Krypton** | 网络栈优化，减少延迟和带宽占用 |
| **FerriteCore** | 内存布局优化，减少内存占用 |
| **ModernFix** | 综合性能修复，修复多种原版性能问题 |

这套方案的优势在于：
1. 获得接近 Paper 的性能提升
2. 保持原版兼容性，Mod 兼容性好
3. 按需选择优化模组
4. 可以同时运行少量功能 Mod

## 适用场景

- **轻量优化原版服** — 用 Fabric + 优化模组代替 Paper，保持原版体验
- **轻量 Mod 服** — 少量技术 Mod + 优化模组
- **需要快速更新版本的服务器** — Fabric 对新版本的适配速度最快
- **中小型服务器** — 性能和兼容性的最佳平衡点

## 局限性

- 大型整合包的生态不如 Forge 丰富
- Fabric API 的功能不如 Forge 完善（仍在快速发展中）
- 部分大型 Mod（如 Thermal Series、Twilight Forest 等）仍在 Forge 独占
