# Purpur

> 基于 Paper 的 Minecraft 服务端核心，以丰富的游戏机制自定义选项著称。

## 概述

Purpur 是 Minecraft Java Edition 的一个服务端核心（Server Software），基于 **Paper** 发展而来，兼容 Bukkit/Spigot/Paper 的整个插件生态。它在 Paper 的高性能基础上，额外提供了大量可配置的**游戏机制调整选项**，让服主无需安装插件就能修改很多游戏行为。

## 关键信息

| 属性 | 内容 |
|------|------|
| **类型** | Minecraft 服务端核心（基于 Paper） |

| **性能基线** | 与 Paper 相当（~1.8x 原生） |
| **独特优势** | 大量可定制的游戏机制选项（Purpur Config） |

## 核心特色

Purpur 区别于其他 Paper Fork 的最大特点是其 **Purpur Config** 系统，允许在不安装任何插件的情况下调整超过 200 项游戏机制：

- 修改玩家行为（允许/禁止飞行、 Elytra 火箭加速、爬墙等）
- 调整实体 AI（村民工作速度、动物生长时间等）
- 自定义堆叠上限（潜影盒可堆叠、药剂可堆叠等）
- 红石机制调整（活塞推动上限、漏斗速率等）
- 世界边界默认为无限（无需额外插件）

## 适用场景

- **插件生存服** — 最主流的用途，Paper 的稳定性 + Purpur 的灵活性
- **需要定制游戏体验的场景** — 用 Purpur Config 代替插件做简单修改，减少插件数量
- **中小型 Java 版服务器** — 单服模式下的绝佳选择

## 局限性

- 仍然是单线程主循环架构，受限于 Paper 的整体设计
- 如果不需要定制游戏机制，直接使用 Paper 即可获得同样的性能
