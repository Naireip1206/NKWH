# MC服务器搭建思路摘要

> 来源：[[MC服务器搭建的整体思路]] — 用户撰写的一篇 MC 服务器搭建综合指南。

## 概述

本文是关于搭建 Minecraft Java Edition 服务器的综合指南，覆盖了从硬件选型到运维维护的全流程。核心观点是：AI 工具虽然降低了搭建门槛，但理解底层原理才能更好地运营和维护服务器。

## 关键要点

### 硬件选型
- MC 服务器的瓶颈主要在 **CPU 单核性能** 和 **内存容量**
- CPU 单核性能决定玩家承载能力，核心数的优先级较低
- 内存需求随玩家人数和 Mod/插件数量线性增长
- NVMe SSD 对区块加载速度有明显提升

### 环境准备
- Java 版本必须与 MC 版本严格对应（Java 8 / 17 / 21）
- Linux（Ubuntu/Debian）是推荐的服务器操作系统
- 不同服务端核心（Paper、Fabric、Forge）适用于不同场景

### 启动优化
- Aikar's Flags 是业界公认的 JVM 优化参数集合
- G1GC 是 MC 服务器最适合的垃圾回收器
- 生产环境推荐使用 Systemd 管理服务器进程

### 网络与安全
- 家庭自建服需要配置端口转发
- 推荐使用 TCPShield 或 Cloudflare 进行 DDoS 防护
- 大型服务器可采用反代 → 代理 → 后端的分层架构

### 运维
- 定期备份（crontab + tar）是生存线
- TPS 和 MSPT 是核心性能指标
- Chunky 预生成区块可显著减少运行时的性能抖动
