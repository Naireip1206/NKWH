# LLM Wiki 方法论 — 来源摘要

> **原文**: `llm-wiki.md`（已归档至 [[原始资料/来源/LLM Wiki 方法论文档]]）

## 核心思想

传统的 RAG（检索增强生成）方式是在每次查询时重新发现和拼凑知识，没有积累效应。LLM Wiki 的核心理念是：由 LLM **增量式构建并维护一个持久的维基知识库**——一个结构化的、相互链接的 Markdown 文件集合，位于用户和原始资料之间。

**关键区别：维基是一个持久的、不断叠加的产物。** 交叉引用已经存在，矛盾已经被标记，综合已经反映了所有阅读过的内容。每次添加新来源和提出新问题，知识库都变得更丰富。

## 三层架构

| 层级 | 内容 | 谁拥有 |
|------|------|--------|
| **原始资料** | 文章、论文、数据文件（不可变） | 用户策展 |
| **知识库（Wiki）** | LLM 生成的 Markdown 页面 | LLM 全权维护 |
| **模式文档（Schema）** | 定义结构、约定和流程的配置文件 | 用户与 LLM 共同演化 |

## 三种核心操作

1. **摄取（Ingest）** — 放入新来源 → LLM 读取、讨论、写摘要、更新索引和相关页面、记录日志。一个来源可能触及 10-15 个页面。
2. **查询（Query）** — 提问 → LLM 搜索页面、综合回答、将有价值的回答归档为新页面。
3. **维护（Lint）** — 健康检查 → 查找矛盾、过时声明、孤立页面、缺失页面和缺失引用。

## 两个关键文件

- **`索引.md`** — 内容导向的目录（按类别组织，每次摄取后更新）
- **`日志.md`** — 时间顺序的记录（每条以统一前缀开头，便于搜索）

## 为什么有效

维护知识库最繁琐的不是阅读或思考，而是**簿记工作**——更新交叉引用、保持摘要最新、标记矛盾。人类因为维护负担增长快于价值而放弃维基。LLM 不会厌倦、不会忘记更新引用、可以一次处理 15 个文件。维护成本接近于零。

人类的工作是：策展来源、指导分析、提出好问题、思考意义。LLM 的工作是：其他一切。

## 重要启示

- 本知识库本身就是这个方法论的实例化
- 目录结构、页面格式、工具——都取决于具体领域和个人偏好
- 一切可选且模块化——选择有用的，忽略不需要的
