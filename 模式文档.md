# 模式文档 — LLM Wiki 知识库架构

> 本文档定义了本知识库的结构、约定和工作流程。它是 LLM 维护知识库的核心操作手册。

## 概述

本知识库遵循 **LLM Wiki** 方法论文档（见 [[原始资料/来源/LLM Wiki 方法论文档]]）的核心理念：由 LLM 增量式构建并维护一个持久的、结构化的、相互链接的维基知识库。知识一旦被编译就保持更新，而不是在每次查询时重新推导。

### 三层架构

1. **原始资料层** (`原始资料/`) — 不可变的来源文档集合。LLM 从中读取但永不修改。这是事实源。
2. **知识库层** (`知识库/`) — LLM 生成和维护的 Markdown 文件。LLM 完全拥有这一层：创建页面、在新来源到达时更新、维护交叉引用、保持一致性。
3. **模式文档**（本文档）— 定义知识库结构、约定和工作流程。LLM 和用户共同演化它。

## 目录结构

```
原始资料/                    # 不可变的原始来源
├── 资产/                   # 图片等附件资源
├── 来源/                   # 已归档的来源文档（可保留原文件链接）
└── README.md              # 原始资料索引

知识库/                     # LLM 生成和维护的 Wiki
├── 索引.md                # ★ 内容目录（索引）
├── 日志.md                # ★ 操作日志（按时间顺序）
├── 概览.md                # 全局概览与综合论述
├── 实体/                  # 实体页面（人物、组织、项目等）
├── 概念/                  # 概念页面（理论、方法、术语等）
├── 来源/                  # 来源摘要页面
├── 专题/                  # 专题深度研究
└── 比较/                  # 对比分析页面

模式文档.md                 # ★ 本文档 — 架构模式文档
```

## 页面约定

### YAML Frontmatter

每个知识库页面应包含 YAML 前置元数据：

```yaml
---
标题: 页面标题
标签: [标签1, 标签2]
创建日期: 2026-07-17
来源数量: 0
相关实体: []
相关概念: []
类型: 实体 | 概念 | 来源摘要 | 比较 | 专题
状态: 活跃 | 待更新 | 待补充
---
```

### 页面格式

- 使用 `#` 一级标题作为页面标题
- 使用 `##` 和 `###` 组织层级
- 使用 `[[内部链接]]` 链接知识库中的其他页面
- 使用 `![[图片]]` 嵌入本地图片
- 使用 `> 引用` 标注引用
- 使用 `---` 分隔不同章节

### 命名规范

- 页面名称为中文，简明扼要
- 文件名与标题一致
- 避免过长的文件名
- 实体页面以实体名称命名
- 概念页面以概念名称命名

## 工作流程

### 1. 摄取流程 (Ingest)

当用户添加新来源时，LLM 应执行以下步骤：

1. **读取来源** — 完整阅读原始资料
2. **讨论要点** — 与用户讨论关键收获和重点方向
3. **撰写来源摘要** — 在 `知识库/来源/` 下创建摘要页面
4. **更新索引** — 在 [[知识库/索引.md]] 中添加条目
5. **更新相关页面** — 更新涉及的所有实体、概念页面（可能跨 10-15 个页面）
6. **记录日志** — 在 [[知识库/日志.md]] 中追加条目
7. **检查一致性** — 标记与现有内容矛盾或互补之处

一次处理一个来源，用户保持参与以指导重点方向。

### 2. 查询流程 (Query)

当用户提问时，LLM 应：

1. **查阅索引** — 先读取 [[知识库/索引.md]] 定位相关页面
2. **深入阅读** — 读取找到的页面获取详细信息
3. **综合回答** — 整合多页面信息，给出带引用的回答
4. **归档价值回答** — 有价值的回答应作为新页面归档到知识库中
   - 对比分析 → `知识库/比较/`
   - 专题研究 → `知识库/专题/`
   - 新的联系和发现 → 相应概念或实体页面

### 3. 维护流程 (Lint)

定期健康检查，LLM 应检查：

- **矛盾** — 页面之间的冲突声明
- **过时** — 已被新来源取代的旧主张
- **孤立页面** — 没有入站链接的页面
- **缺失页面** — 被频繁提及但尚无独立页面的重要概念
- **缺失引用** — 缺少的交叉引用
- **数据缺口** — 可通过网络搜索填充的信息空白
- **索引同步** — 索引是否与实际页面列表一致

## 索引与日志

### 索引文件：[[知识库/索引.md]]

内容导向的目录。每页一条，包含链接、一句话摘要和元数据。按类别组织（实体、概念、来源等）。每次摄取后更新。

### 日志文件：[[知识库/日志.md]]

时间顺序的记录。每次操作的追加记录。每条日志以统一前缀开头：

```
## [2026-07-17] 摄取 | 文章标题
## [2026-07-17] 查询 | 用户问题简述
## [2026-07-17] 维护 | 健康检查
```

方便使用 Unix 工具或搜索快速查看最近操作。

## 标签系统

- `#实体` — 人物、组织、项目、产品
- `#概念` — 理论、方法、术语、框架
- `#来源` — 文章、书籍、论文、报告
- `#专题` — 研究话题、深度领域
- `#待更新` — 需要补充或修订的页面
- `#待探索` — 值得进一步研究的方向

## 工具与集成

- **Obsidian** — 知识库的 IDE，用于浏览、链接和图谱视图
- **Obsidian Web Clipper** — 浏览器扩展，将网页转为 Markdown 存入原始资料
- **Dataview 插件** — 基于 YAML frontmatter 运行查询，生成动态表格和列表
- **Marp 插件** — 基于 Markdown 的幻灯片格式，用于知识展示
- **Git** — 版本历史、分支和协作（知识库就是 Markdown 的 Git 仓库）

---

## 演化原则

- 本文档（模式文档）由用户和 LLM 共同演化
- 随着领域发展，持续优化目录结构、页面格式和工作流程
- 对当前工作流不适用的部分可以移除，需要的新部分可以添加
- 一切可选且模块化 — 选择有用的，忽略不需要的
